Installation

1
Voraussetzungen prüfen

Folgende Software muss installiert sein:

  • PHP 8.1 oder höher
  • XAMPP (Apache + MariaDB) oder anderer Webserver
  • Composer (PHP Paketmanager)
2
Projekt-Dateien platzieren

Den slim-api Ordner nach C:\xampp\htdocs\ kopieren:

C:\xampp\htdocs\
└── slim-api\
    ├── composer.json
    ├── config\
    ├── public\        ← Apache DocumentRoot
    └── src\
3
Abhängigkeiten installieren

Terminal im slim-api Ordner öffnen:

cd C:\xampp\htdocs\slim-api
composer install

Dies erstellt den vendor/ Ordner mit Slim Framework und allen Abhängigkeiten.

4
Datenbank konfigurieren

SQL-Dump einspielen:

mysql -u root -p handwerkerpro_db < handwerkerpro_db.sql

Dann in config/database.php die Zugangsdaten anpassen:

return [
    'host' => 'localhost',
    'name' => 'handwerkerpro_db',
    'user' => 'root',
    'pass' => '',  // Dein Passwort
];
5
Apache konfigurieren

In C:\xampp\apache\conf\extra\httpd-vhosts.conf eintragen:

<VirtualHost *:80>
    DocumentRoot "C:/xampp/htdocs/slim-api/public"
    ServerName localhost
    <Directory "C:/xampp/htdocs/slim-api/public">
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>

In C:\xampp\apache\conf\httpd.conf sicherstellen dass diese Zeile aktiv ist:

LoadModule rewrite_module modules/mod_rewrite.so
6
Apache neu starten & testen

XAMPP Control Panel → Apache → Restart

Im Browser öffnen:

http://localhost/api

Wenn eine JSON-Antwort mit allen Endpoints erscheint, ist die Installation erfolgreich.

7
Test-Client öffnen
http://localhost/docs/testclient.html

Authentifizierung

Die API unterstützt zwei Methoden. Methode 1 (API-Key) wird für alle Anfragen empfohlen.

Methode 1: API-Key Header
X-API-Key: admin-key-12345

// Alternativ:
Authorization: Bearer admin-key-12345
Methode 2: Session-Login
POST /api/auth/login
{
  "email":    "[email protected]",
  "password": "passwort"
}

admin-key-12345 Max Mustermann admin Vollzugriff
meister-key-67890 Lena Zimmer meister Lesen, Erstellen, Bearbeiten, Löschen (außer Material löschen)
geselle-key-11111 Klaus Bauer geselle Lesen, Erstellen, Bearbeiten
Rollenhierarchie
// Zugriffslevel (höher = mehr Rechte)
admin (5) > meister (4) > geselle (3) > azubi (2) > buero (1)

// Wer darf was:
GET    (alle Rollen)  → Daten lesen
POST   (geselle+)    → Neue Einträge erstellen
PUT    (geselle+)    → Einträge bearbeiten
DELETE (meister+)   → Kunden/Aufträge löschen
DELETE (admin)      → Material löschen

Fehlerbehandlung & Validierung

Validierungsregeln
  • Pflichtfelder werden geprüft → 422
  • ENUM-Werte gegen Whitelist geprüft → 422
  • Datumsformat YYYY-MM-DD → 422
  • Numerische Werte auf Positivität → 422
  • E-Mail via filter_var() → 422
  • Ressource nicht gefunden → 404
  • DB-Trigger-Fehler (z.B. offene Aufträge) → 409
Beispiel 422-Antwort
{
  "status":  "error",
  "message": "Pflichtfelder fehlen.",
  "details": {
    "fehlende_felder": [
      "vorname",
      "nachname"
    ]
  }
}

Sicherheit

Prepared Statements

Alle DB-Abfragen über PDO Prepared Statements. String-Concatenation für SQL ist vollständig verboten.

Passwort-Hashing

Argon2ID (Standard) oder bcrypt via password_hash(). Passwörter werden niemals im Klartext gespeichert.

Input-Sanitizing

Alle Eingaben werden via strip_tags() und trim() bereinigt (XSS-Schutz).

CORS-Header

CorsMiddleware setzt alle notwendigen Header. In Produktion sollte Allow-Origin eingeschränkt werden.

Dashboard

GET /api/dashboard alle Rollen

Vollständige Dashboard-Übersicht mit KPIs, heutigen Terminen, Notfall-Aufträgen und kritischen Lagerbeständen.

Beispiel-Aufruf
curl http://localhost/api/dashboard \
  -H "X-API-Key: admin-key-12345"
Enthaltene Daten
  • kpis — Kunden gesamt, offene Aufträge, überfällige Rechnungen, offener Betrag, kritische Lagerbestände
  • auftraege_statistik — Anzahl pro Status
  • rechnungen_statistik — Anzahl pro Status + offener Betrag
  • termine_heute — Alle Termine des heutigen Tages
  • notfall_auftraege — Offene Notfall/Dringend Aufträge
  • lager_kritisch — Materialien mit Bestand < 5
  • wartende_kunden — Kunden mit Status "neu" (Top 5)
  • mitarbeiter_auslastung — Geplante Stunden diese Woche

PUT /api/auftraege/{id}/status geselle+

Status ändern. Spezialverhalten:

  • abgeschlossen → ruft sp_auftrag_abschliessen auf → Rechnung wird automatisch erstellt
  • storniert → ruft sp_auftrag_stornieren auf → Materialien werden ins Lager zurückgebucht
curl -X PUT http://localhost/api/auftraege/3/status \
  -H "X-API-Key: admin-key-12345" \
  -H "Content-Type: application/json" \
  -d '{ "status": "abgeschlossen" }'
DELETE /api/auftraege/{id} meister+

Löscht Auftrag. Bereits abgerechnete Aufträge können nicht gelöscht werden → 409.

curl -X DELETE http://localhost/api/auftraege/15 \
  -H "X-API-Key: meister-key-67890"
